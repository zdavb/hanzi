# 产品需求文档 (PRD) - 萌芽汉字启蒙 Web 平台 (Moe Hanzi)

## 1. 项目概述 (Project Overview)

### 1.1 背景
目标用户为一名4岁的幼儿园小班女孩。家长希望对其进行汉字启蒙，范围涵盖中国大陆小学一年级要求的汉字。由于孩子年龄较小，系统设计需侧重于“兴趣激发”而非“枯燥记忆”，采用 Web 形式方便在平板或电脑大屏上使用，保护视力。

### 1.2 目标
- **核心目标**：让孩子在轻松有趣的环境中，循序渐进地习得小学一年级基础汉字（约 600+ 字）。
- **体验目标**：通过严格的分级体系和即时的正向反馈，让孩子保持学习动力。
- **形式目标**：构建一个交互友好、视觉吸引力强的 Web 应用，支持全量汉字的动画/图解展示。

## 2. 用户画像 (User Persona)

### 2.1 核心用户：萌萌 (化名)
- **年龄**：4岁 (幼儿园小班)
- **特征**：
  - 注意力集中时间短 (约 10-15 分钟)。
  - 处于形象思维阶段，对图形记忆优于抽象符号记忆。
  - 需要即时的正向反馈 (鼓励、动画效果)。
  - 操作能力有限，需要极大且直观的按钮。

### 2.2 辅助用户：家长
- **需求**：
  - 内容需严格对标小学一年级标准，无遗漏。
  - 希望能直观看到孩子的学习进度。
  - 操作简便，打开浏览器即可用。

## 3. 内容策略 (Content Strategy)

### 3.1 选字与分级体系
严格按照 `hanzi_level_list.md` 整理的 5 个层级进行教学，共计约 600+ 字：
- **Level 1: 萌芽期 (50字)**：建立“字”与“实物/图形”的直接联系 (数字、自然、身体、核心称谓)。
- **Level 2: 探索期 (120字)**：认识周围的世界 (色彩、动物、植物、天气、时间)。
- **Level 3: 表达期 (150字)**：学会表达动作、感受及状态 (动作、感官、形容词)。
- **Level 4: 社会期 (160字)**：进入社会化场景 (学校、家庭、交通、亲属)。
- **Level 5: 进阶期 (120字)**：完善抽象概念 (心理、逻辑、虚词、姓氏)。

### 3.2 汉字卡片结构 (Card Structure)
每一个汉字的学习单元 (Card) **必须** 包含以下四个核心元素（内容需严格对标 `hanzi_content_detailed.md`）：
1.  **汉字 (Hanzi)**：屏幕中央展示标准的楷体/手写体汉字。
2.  **拼音 (Pinyin)**：标注标准拼音及声调。
3.  **英文 (English)**：对应的英文含义，帮助建立双语联系。
4.  **动画/插画 (Animation/Illustration)**：
    - **数据来源**：所有动画逻辑和样式代码来自于 `hanzi_demo.html` 和 `hanzi_data.js`。
    - **核心字**：使用 CSS/SVG 动画展示字形演变或动态联想 (如“雨”字有雨滴落下)。
    - **通用字**：使用 Emoji 或静态插画辅助记忆。

## 4. 功能需求 (Functional Requirements)

### 4.1 核心交互与导航 (Core Interaction)
- **卡片式学习流**：
  - 界面主体为一张巨大的汉字卡片。
- **切换导航**：
  - **上一页 (Previous)**：左侧或底部左边大按钮，允许返回复习。
  - **下一页 (Next)**：右侧或底部右边大按钮，进入下一个新字。
  - 支持键盘左右方向键切换 (桌面端优化)。

### 4.2 进度反馈系统 (Progress Tracking)
- **顶部进度条 (Top Progress Bar)**：
  - 屏幕顶部常驻显示进度条。
  - 实时显示当前进度 (例如：Level 1 - 5/50)。
  - 视觉上使用鲜艳的颜色 (如绿色或橙色) 填充。
- **计数器**：
  - 清晰显示“已学会 / 总字数” (例如：已认识 12 个字)。

### 4.3 激励与反馈 (Gamification & Rewards)
- **里程碑奖励 (Milestone Animations)**：
  - **触发机制**：每学习完成一定数量的汉字 (例如每 5 个或 10 个字)。
  - **表现形式**：
    - 屏幕弹出全屏动画 (如烟花、气球升空、可爱动物跳舞)。
    - 伴随“太棒了！”、“真厉害！”的语音或文字提示。
    - 暂停 3-5 秒，让孩子享受成就感，然后自动或手动返回学习界面。

### 4.4 数据与状态保持
- **自动存档**：
  - 浏览器 LocalStorage 自动记录最后一次学习的汉字索引。
  - 再次打开应用时，自动跳转到上次的学习进度，无需从头开始。

## 5. UI/UX 设计原则

1.  **极简主义**：
    - 屏幕上除了卡片、导航和进度条，不应有干扰元素。
    - 背景色应柔和 (米黄、淡蓝)，保护视力。
2.  **超大触控区**：
    - “上一个”、“下一个”按钮直径至少 60px (移动端) 或占据屏幕边缘明显区域。
3.  **视觉层级**：
    - 汉字 (最大) > 动画 > 拼音/英文 > 导航按钮。

## 6. 技术栈 (Tech Stack)

- **前端框架**：HTML5 + CSS3 + Vanilla JavaScript (轻量级，无需复杂构建工具)。
- **数据源**：`hanzi_data.js` (JSON 格式存储所有 600+ 汉字的元数据)。
- **动画实现**：
  - **CSS Animations**：用于关键汉字的笔画或形意动画。
  - **Emoji/SVG**：用于通用汉字的配图。

## 7. 开发路线图 (Roadmap)

- **Step 1**: 完善 `hanzi_data.js`，确保包含所有 600+ 字的完整数据 (字、拼音、英文、插画类型)。
- **Step 2**: 更新 Web 界面，实现“上一个/下一个”导航逻辑和进度条渲染。
- **Step 3**: 集成里程碑动画 (Confetti/Fireworks 效果)。
- **Step 4**: 优化 CSS 样式，确保在 iPad/PC 端均有良好表现。
